## Overview
- **Priority:** High

- **Status:** Completed

Server-side Telegram OIDC: code exchange, JWT verification, user creation, Supabase session generation.

## Key Insights
- Telegram OIDC endpoints: `oauth.telegram.org/auth`, `/token`, `/.well-known/jwks.json`
- JWT claims: `sub` (telegramId), `name`, `preferred_username`, `picture` — NO email
- Session via: `admin.createUser()` → `admin.generateLink()` → `verifyOtp()` = Supabase session
- Placeholder email: `tg_{telegramId}@tg.clawquest.ai`

## Implementation Steps

### 1. Create `telegram-oidc.utils.ts`

OIDC utility functions:

```typescript
// Fetch JWKS from https://oauth.telegram.org/.well-known/jwks.json
// Cache keys in memory (refresh every 24h)
export async function getTelegramJwks(): Promise<JsonWebKeySet>

// Verify Telegram ID token signature + claims
// Checks: iss, aud (= bot_id), exp, iat
// Returns decoded payload: { sub, name, preferred_username, picture }
export async function verifyTelegramIdToken(idToken: string): Promise<TelegramClaims>

// Exchange authorization code for tokens via POST oauth.telegram.org/token
export async function exchangeTelegramCode(params: {
  code: string
  codeVerifier: string
  redirectUri: string
}): Promise<{ id_token: string; access_token: string }>
```

Use `jose` library for JWT verification (already common in Node ecosystem). Install: `pnpm --filter api add jose`.

### 2. Create `telegram-auth.service.ts`

Core business logic:

```typescript
export async function handleTelegramLogin(params: {
  code: string
  codeVerifier: string
  redirectUri: string
  prisma: PrismaClient
  supabaseAdmin: SupabaseClient
}): Promise<{
  session: { access_token: string; refresh_token: string }
  user: User
  isNewUser: boolean
  linkedAgents: string[]
}>
```

Flow:
1. `exchangeTelegramCode()` → get id_token
2. `verifyTelegramIdToken()` → get claims (telegramId, name, username)
3. Find existing user by `telegramId`
4. If not found → create new:
   a. `supabaseAdmin.auth.admin.createUser({ email: placeholder, email_confirm: true, user_metadata: { telegramId, provider: 'telegram' } })`
   b. `prisma.user.create({ ... telegramId, telegramUsername, supabaseId })`
5. Auto-link agents: query `TelegramLink` by `telegramId`, update `Agent.ownerId` where null
6. Generate session:
   a. `supabaseAdmin.auth.admin.generateLink({ type: 'magiclink', email: user.email })`
   b. `supabaseAdmin.auth.verifyOtp({ token_hash: hashed_token, type: 'magiclink' })`
7. Return session + user + linkedAgents

### 3. Add routes to `auth.routes.ts`

**`POST /auth/telegram`** — Login/register via Telegram (public)
```
Body: { code: string, codeVerifier: string, redirectUri: string }
Response: { session, user, isNewUser, linkedAgents }
```

**`POST /auth/telegram/link`** — Link Telegram to existing account (protected)
```
Body: { code: string, codeVerifier: string, redirectUri: string }
Response: { user, linkedAgents }
```

Link endpoint: verify JWT → check telegramId not already linked to another user → update user.telegramId + telegramUsername → auto-link agents → return.

## Security
- Server-side code exchange (client_secret never on frontend)
- JWT verified via JWKS (public key validation)
- Claims validated: iss = `https://oauth.telegram.org`, aud = bot_id, exp not past
- State parameter validated (generated by frontend, stored in sessionStorage)
