## Key Insights

- Follows existing pattern: `telegramId` + `telegramUsername` in User model
- xId/discordId are `@unique` â€” one account per user, and a given social account can only link to one ClawQuest user
- `xHandle` / `discordHandle` do NOT need `@unique` â€” handles can change, ID is the stable identifier
- Migration is additive (nullable fields) â€” zero downtime, no data loss

- Add `xHandle String?` â€” Twitter/X username (from `identity_data.user_name`)
- Add `discordId String? @unique` â€” Discord user ID
- Add `discordHandle String?` â€” Discord username
