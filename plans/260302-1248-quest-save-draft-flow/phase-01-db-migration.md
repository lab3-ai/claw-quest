# Phase 1: DB Migration

## Context
- Brainstorm: `plans/reports/brainstorm-260302-1248-quest-save-draft-flow.md`
- Schema: `apps/api/prisma/schema.prisma`

## Overview
- **Priority:** high
- **Status:** completed
- **Effort:** small (~15 min)

Add missing fields and set `rewardAmount` default to allow partial draft saves.

## Key Insights

- `rewardAmount` has NO default → blocks saving drafts without reward info
- `network` (blockchain) captured in form but never persisted
- `drawTime` (Lucky Draw deadline) captured but never sent/stored
- All changes are additive — no data backfill needed

## Requirements

### Functional
- `rewardAmount` default to 0 (allows draft without reward)
- `network` optional field (stores selected blockchain name)
- `drawTime` optional field (Lucky Draw specific deadline)

### Non-functional
- Zero-downtime migration (all nullable/default)
- No breaking changes to existing data

## Related Code Files

**Modify:**
- `apps/api/prisma/schema.prisma` — Quest model

**Creates:**
- `apps/api/prisma/migrations/YYYYMMDD_add_quest_draft_fields/migration.sql` (auto-generated by prisma)

## Implementation Steps

1. Update Quest model in `schema.prisma`:
   ```prisma
   rewardAmount  Int       @default(0)     // add @default(0)
   network       String?                   // new field
   drawTime      DateTime?                 // new field
   ```

2. Run migration:
   ```bash
   cd apps/api && npx prisma migrate dev --name add_quest_draft_fields
   ```

3. Verify migration SQL is correct (should be 3 ALTERs, no data changes)

4. Run `npx prisma generate` to update client types
