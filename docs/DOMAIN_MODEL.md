# Domain Model

## 🧬 Entities & Value Objects

### 1. Human (Owner)
- The root entity.
- Can own multiple Agents (future proofing), but MVP limits to 1.
- Holds auth credentials.

### 2. Agent
- The autonomous actor.
- **Invariant**: Must have an Owner.
- **Invariant**: Can have at most 1 active TelegramLink.
- **State Machine**:
    - `IDLE`: Ready for commands.
    - `QUESTING`: Busy, cannot accept new quests.
    - `OFFLINE`: (System state) Webhook failures or deactivated.

### 3. Quest
- A blueprint for work.
- Defined in code or DB (Catalog).
- **Properties**: Duration, Success Chance, Rewards.

### 4. Activation Code
- Ephemeral secret generated by the Dashboard.
- Used to prove ownership when talking to the Telegram Bot.
- **Format**: 6-digit alphanumeric or short words.
- **TTL**: 15 minutes (Security Best Practice).

## 🔄 Relationships

- **Human 1:N Agent** (One human owns N agents)
- **Agent 1:1 TelegramLink** (One agent is linked to exactly one TG account)
- **Agent 1:N AgentLog** (History of actions)

## 🧠 Business Logic Invariants
1. A Telegram user cannot link to an Agent if they are already linked to another one (unless they unlink first).
2. An Agent cannot start a quest if `status != "idle"`.
3. Quest completion must trigger a transactional update:
    - Log entry created.
    - Agent status set to `IDLE`.
    - Rewards applied.
